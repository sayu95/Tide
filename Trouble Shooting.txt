-Docker 명령어
작성한 docker-compose 파일로 컨테이너 생성 실행
docker-compose up -d

-프로젝트 루트에 있는 .env 파일에 정의된 Key=Value 쌍을 읽어 파이썬의 **시스템 환경변수(os.environ)**로 로드해주는 라이브러리입니다.
목적 (Why?): 1.  보안(Security): DB 비밀번호, API 키 등 민감 정보를 소스코드(settings.py)에서 분리.
2.  환경 분리(Environment Isolation): 로컬(Local), 테스트(Test), 운영(Prod) 서버마다 각기 다른 설정을 코드 수정 없이 적용.
import os
from dotenv import load_dotenv

# 1. 기본 로드 (가장 많이 사용)
load_dotenv()

# 2. 특정 경로 지정 (루트가 아닐 때)
# load_dotenv(dotenv_path=".env.dev")

# 3. 값 가져오기
db_password = os.getenv('DB_PASSWORD')

# 4. 기본값 설정 (자바의 @Value("${prop:default}") 와 동일)
db_port = os.getenv('DB_PORT', '5432')

-indicators_indicator	정선회님이 직접 만든 테이블입니다.	Indicator Entity 클래스
auth_user	사용자(ID/PW) 정보가 저장되는 곳입니다.	Spring Security의 UserDetails
auth_group / permission	사용자 권한 및 그룹 관리 테이블입니다.	Role-Based Access Control (RBAC)
django_migrations	가장 중요한 곳! 어떤 마이그레이션 파일이 실행됐는지 기록합니다.	flyway_schema_history
django_session	로그인한 사용자의 세션 정보가 담깁니다.	Spring Session / Redis 역할
django_content_type	장고가 관리하는 모델(테이블)들의 메타데이터입니다.	Java Reflection 정보의 DB판
